Usage Guide

Rusty Requester is a local-first API client for building requests, inspecting responses, running collections, and sharing safe snippets without an account or cloud sync.

First request

  1. Create a collection from the sidebar.
  2. Add a request inside that collection.
  3. Choose a method, enter a URL, then add params, headers, cookies, body, auth, or tests.
  4. Click Send, or press Cmd/Ctrl + Enter.

Saved requests persist in the local workspace file. Draft tabs persist until you save or close them.

Collections

Organize requests

Use collections and nested folders for APIs, projects, or environments. Folder overview pages show recursive request counts and descriptions.

OpenAPI and Postman import

Import JSON, YAML, OpenAPI 3.x, or Postman Collection v2.1 files. OpenAPI imports create generated request metadata for later refresh flows.

Environments

Define variables such as base_url, tenant, or token, then reference them as {{base_url}} in URLs, headers, bodies, and auth fields.

Use Environment Compare to pick a source and target environment, inspect added, missing, changed, and unchanged keys, copy a masked summary, and add missing keys to the target.

Auth

Requests support no auth, Bearer token, Basic auth, and OAuth 2.0 Authorization Code + PKCE. OAuth tokens are stored locally with the workspace, so treat exports and backups as sensitive unless you use a redacted export mode.

Responses

Response views include JSON, Tree, Preview, Events for SSE streams, Diff, and Raw. Status, timing, request/response byte sizes, body search, copying, and save-to-file controls are available from the response toolbar.

Collection Runner

Run all collections or one folder with optional CSV/JSON data rows. Runner-scoped variables overlay the active environment for each row. Results stream live with status, timing, assertion counts, extractor misses, and selectable detail rows.

Save runner presets when you repeat the same smoke or regression flow. Presets store scope, selected environment id, and visible data rows.

Import and export

  • Paste cURL: paste a cURL command into the dialog to create a request.
  • Collection export: export all collections as JSON or YAML.
  • Git workspace format: write a deterministic directory with a manifest and one readable request JSON file per request. See Git workspace format.

Safe sharing

Snippet sharing is redacted by default for cURL, Python, JavaScript fetch, and HTTPie. Redaction preserves request shape while masking obvious authorization, cookie, sensitive query, and sensitive body values. Use Copy raw only when you intentionally need the unredacted snippet.

Collection exports run a local-only secret scan. If likely secrets are found, you can cancel, export original content, or export a redacted copy. Nothing is uploaded.

Secret scanning is heuristic. It catches common patterns and sensitive key names, but unusual secret formats can be missed and placeholder examples can produce false positives.

Backups

Rusty Requester can create workspace backups before risky operations such as imports. Use the Backups window to create, inspect, and restore local snapshots. Restoring creates a current-state backup first.

Shortcuts

  • Cmd/Ctrl + Enter send request
  • Cmd/Ctrl + N new request
  • Cmd/Ctrl + S save draft
  • Cmd/Ctrl + K focus sidebar search
  • Cmd/Ctrl + P command palette
  • Cmd/Ctrl + Shift + P actions palette
